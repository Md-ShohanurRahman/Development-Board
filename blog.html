Question-1: What are the different ways to select an element in the DOM?

-----------------------------------------------------------------------------------------------------------------------------------------

There are several ways to select an element in the DOM (Document Object Model). The most common methods include:

  1. 1.getElementById(): Selects an element by its unique ID.
  2. 2.getElementById(): Selects an element by its unique ID.
  3. 3.getElementsByClassName(): Selects elements with a specific class name.
  4. 4.getElementsByTagName(): Selects elements with a specific tag name.
  5. 5.querySelector(): Selects the first element that matches a CSS selector.
  6. These methods allow you to target elements based on their ID, class, tag, or CSS selector.

Question-2: What is the difference between innerHTML, innerText, and textContent ?

----------------------------------------------------------------------------------------------------------------------------------------

The difference between innerHTML, innerText, and textContent lies in how they handle the content of an element:

  1. 1.innerHTML: Returns or sets the HTML content inside an element, including any HTML tags. It can also modify the HTML structure, such as adding or removing elements.
  2. 2.innerText: Returns or sets the visible text inside an element, excluding any HTML tags. It takes CSS styling into account, such as hidden text or text hidden by display: none.
  3. 3.textContent: Returns or sets the text content inside an element, but unlike innerText, it does not take CSS styles into account. It simply gives or changes the raw text, including hidden text.
  4. In summary, innerHTML handles HTML, innerText focuses on visible text, and textContent deals with all text, visible or not.

Question-3: What is event delegation in the DOM?

----------------------------------------------------------------------------------------------------------------------------------------

Event delegation in the DOM is a technique where instead of attaching event listeners to individual elements, you attach a single event listener to a parent element. This parent element listens for events on its child elements. When an event occurs on a child element, it "bubbles" up to the parent, which can then handle the event.This approach is more efficient because it reduces the number of event listeners in the DOM, especially when there are many child elements. It also allows you to handle events on dynamically added elements without needing to add event listeners to them separately.

Question-4: What is event bubbling in the DOM?

----------------------------------------------------------------------------------------------------------------------------------------

Event bubbling in the DOM is a concept where, when an event is triggered on an element, it doesn't just affect that element. Instead, the event "bubbles up" from the target element to its parent elements, then to their parents, and so on, all the way to the root of the document. For example, if you click on a button inside a div, the event first triggers on the button and then bubbles up to the div, the body, and the document. This allows parent elements to listen for events on their child elements without attaching separate event listeners to each child.
